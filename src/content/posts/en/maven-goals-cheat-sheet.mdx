## Understanding goal chains

When you run a Maven command like `mvn package`, Maven doesn't only execute that goal, it runs every phase leading up to it, in sequence.

<table>
  <thead>
    <tr>
      <th>Command</th>
      <th>Phases</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <code>mvn compile</code>
      </td>
      <td>
        <code>validate -> compile</code>
      </td>
    </tr>
    <tr>
      <td>
        <code>mvn test</code>
      </td>
      <td>
        <code>validate -> compile -> test</code>
      </td>
    </tr>
    <tr>
      <td>
        <code>mvn package</code>
      </td>
      <td>
        <code>validate -> compile -> test -> package</code>
      </td>
    </tr>
    <tr>
      <td>
        <code>mvn install</code>
      </td>
      <td>
        <code>validate -> compile -> test -> package -> install</code>
      </td>
    </tr>
    <tr>
      <td>
        <code>mvn deploy</code>
      </td>
      <td>
        <code>validate -> compile -> test -> package -> install -> deploy</code>
      </td>
    </tr>
    <tr>
      <td>
        <code>mvn verify</code>
      </td>
      <td>
        <code>validate -> compile -> test -> verify</code>
      </td>
    </tr>
  </tbody>
</table>

Each phase is bound by default to one or more plugin goals.  
Example:  
`compile` -> `compiler:compile`  
`package` -> `jar:jar` or `war:war` (depending on packaging type).

## Custom goal bindings

You can attach plugin goals to different phases in your `pom.xml`:

```xml
<plugin>
  <groupId>org.apache.maven.plugins</groupId>
  <artifactId>maven-antrun-plugin</artifactId>
  <executions>
    <execution>
      <phase>generate-sources</phase>
      <goals>
        <goal>run</goal>
      </goals>
    </execution>
  </executions>
</plugin>
```
