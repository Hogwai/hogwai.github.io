## Comprendre les chaînes de goals

Quand vous lancez une commande Maven comme `mvn package`, Maven n'exécute pas uniquement ce goal : il parcourt toutes les phases qui le précèdent, dans l'ordre.

<table>
  <thead>
    <tr>
      <th>Commande</th>
      <th>Phases implicites (exécutées dans l'ordre)</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <code>mvn compile</code>
      </td>
      <td>
        <code>validate -> compile</code>
      </td>
    </tr>
    <tr>
      <td>
        <code>mvn test</code>
      </td>
      <td>
        <code>validate -> compile -> test</code>
      </td>
    </tr>
    <tr>
      <td>
        <code>mvn package</code>
      </td>
      <td>
        <code>validate -> compile -> test -> package</code>
      </td>
    </tr>
    <tr>
      <td>
        <code>mvn install</code>
      </td>
      <td>
        <code>
          validate -> compile -> test -> package -> install
        </code>
      </td>
    </tr>
    <tr>
      <td>
        <code>mvn deploy</code>
      </td>
      <td>
        <code>
          validate -> compile -> test -> package -> install
          -> deploy
        </code>
      </td>
    </tr>
    <tr>
      <td>
        <code>mvn verify</code>
      </td>
      <td>
        <code>validate -> compile -> test -> verify</code>
      </td>
    </tr>
  </tbody>
</table>

Chaque phase est associée par défaut à un ou plusieurs goals de plugin.
Exemple :
`compile` -> `compiler:compile`
`package` -> `jar:jar` ou `war:war` (selon le type de packaging).

## Liaisons de goals personnalisées

Vous pouvez associer des goals de plugin à n'importe quelle phase dans votre `pom.xml` :

```xml
<plugin>
  <groupId>org.apache.maven.plugins</groupId>
  <artifactId>maven-antrun-plugin</artifactId>
  <executions>
    <execution>
      <phase>generate-sources</phase>
      <goals>
        <goal>run</goal>
      </goals>
    </execution>
  </executions>
</plugin>
```
